# Классификация и краткая суммаризация русскоязычных отзывов мобильных приложений

Проект выполнен в рамках финального NLP-проекта ODS AI. Основная цель — автоматизировать анализ русскоязычных отзывов пользователей из Google Play: для каждого отзыва предсказывается его тип и формируется короткое summary с основной мыслью.

Такая система полезна для разработчиков, продуктовых команд и поддержки: вместо ручного просмотра тысяч комментариев можно быстро выделять сообщения об ошибках, запросы новых функций, отзывы о пользовательском опыте и короткие неинформативные оценки.

## Задача

Для каждого отзыва решаются две связанные задачи:

1. **Классификация отзыва** в один из 4 классов:
   - `bug_report` — пользователь сообщает об ошибке, сбое, зависании, потере данных или некорректной работе приложения;
   - `feature_request` — пользователь просит добавить новую функцию, изменить существующую или вернуть удаленную возможность;
   - `user_experience` — пользователь описывает опыт использования, удобство, неудобство, плюсы или минусы без явного сообщения об ошибке и без прямой просьбы о новой функции;
   - `rating` — короткий оценочный отзыв без конкретного содержательного сообщения.

2. **Краткая суммаризация отзыва** — генерация короткой выжимки основной мысли отзыва.

В проекте сравниваются два типа подходов:

- классические модели и BERT-like encoder-модели, решающие только задачу классификации;
- instruction-tuned LLM, которые решают более сложную комплексную задачу: одновременно возвращают класс и краткое summary.

## Датасет

Датасет был собран самостоятельно из русскоязычных отзывов Google Play с помощью Python-скрипта и библиотеки `google-play-scraper`.

В выборку вошли отзывы по 6 популярным приложениям:

- Telegram;
- WhatsApp;
- VK;
- Yandex Go;
- Yandex Maps;
- Avito.

Разметка выполнялась в формате human-in-the-loop:

1. Сначала отзывы были размечены с помощью LLM для получения `label_silver` и `summary_silver`.
2. Для silver-разметки использовались GPT-5.5 и Claude Opus 4.6.
3. Для класса выбирался наиболее часто встречающийся вариант среди предсказаний моделей.
4. Затем была выполнена ручная проверка: исправлялись ошибки классов, некачественные summary и спорные случаи.
5. В финальном обучении и оценке использовались поля `label_gold` и `summary_gold`.

## Подходы

В проекте были протестированы несколько групп моделей.

### 1. Baseline-модели

- `TF-IDF + Logistic Regression` — классический baseline для коротких текстов.
- `CNN` — нейросетевая модель для извлечения локальных n-gram-паттернов.
- `BLSTM` — рекуррентная модель, учитывающая контекст слева и справа.

### 2. BERT-like модели для классификации

BERT-подобные модели решали только задачу классификации отзыва на 4 класса:

- `ruBertLarge`;
- `sbertLarge`;
- `ruRobertaLarge`.

Эти модели оптимальны для sentence classification, поэтому именно они показали лучший результат на чистой задаче классификации.

### 3. LLM для классификации и суммаризации

Для LLM-экспериментов были выбраны 4 instruction-tuned модели:

- `Qwen3-4B-Instruct-2507`;
- `Phi-4-mini-instruct`;
- `Qwen2.5-7B-Instruct`;
- `Mistral-7B-Instruct-v0.3`.

LLM тестировались в двух режимах:

1. **Few-shot inference без дообучения** — модель получает инструкцию и несколько примеров в промпте.
2. **LoRA/QLoRA fine-tuning** — модель дообучается на train-выборке и учится возвращать структурированный ответ с классом и summary.

Искусственный oversampling не использовался: модели сравнивались на исходном размеченном датасете.

## Метрики

Для классификации использовались:

- `Accuracy`;
- `macro-F1`;
- `weighted-F1`.

Для оценки summary использовались:

- `ROUGE-L`;
- `BERTScore`.

Для LLM дополнительно учитывались `parse errors` — случаи, когда модель не вернула корректный структурированный ответ.

## Результаты классификации

| Модель | Accuracy | Macro-F1 | Weighted-F1 |
|---|---:|---:|---:|
| TF-IDF + LogReg | 0.660 | 0.630 | 0.660 |
| CNN | 0.750 | 0.740 | 0.750 |
| BLSTM | 0.730 | 0.650 | 0.730 |
| ruBertLarge | 0.800 | 0.770 | 0.800 |
| sbertLarge | 0.785 | 0.775 | 0.780 |
| **ruRobertaLarge** | **0.822** | **0.801** | **0.824** |

Лучшей моделью для чистой классификации стала `ruRobertaLarge`: `Accuracy = 0.822`, `macro-F1 = 0.801`, `weighted-F1 = 0.824`.

## Результаты LLM без дообучения

| Модель | Parse errors | Accuracy | Macro-F1 | Weighted-F1 | ROUGE-L | BERTScore |
|---|---:|---:|---:|---:|---:|---:|
| **Qwen3-4B-Instruct-2507** | 1 | **0.697** | **0.671** | **0.692** | 0.203 | 0.858 |
| Phi-4-mini-instruct | 0 | 0.629 | 0.593 | 0.625 | 0.160 | 0.853 |
| Qwen2.5-7B-Instruct | 1 | 0.568 | 0.564 | 0.521 | **0.287** | **0.874** |
| Mistral-7B-Instruct-v0.3 | 2 | 0.611 | 0.541 | 0.596 | 0.119 | 0.842 |

Без дообучения LLM уступили BERT-моделям по классификационным метрикам. При этом часть моделей уже демонстрировала способность генерировать осмысленные summary, но иногда нарушала требуемый формат ответа.

## Результаты LLM после LoRA-дообучения

| Модель | Parse errors | Accuracy | Macro-F1 | Weighted-F1 | ROUGE-L | BERTScore |
|---|---:|---:|---:|---:|---:|---:|
| Qwen3-4B-Instruct-2507 | 0 | 0.781 | 0.736 | 0.779 | 0.484 | 0.906 |
| Phi-4-mini-instruct | 0 | 0.775 | 0.693 | 0.768 | 0.439 | 0.901 |
| Qwen2.5-7B-Instruct | 0 | 0.770 | 0.745 | 0.770 | 0.470 | 0.904 |
| **Mistral-7B-Instruct-v0.3** | **0** | **0.802** | **0.747** | **0.799** | **0.552** | **0.920** |

LoRA-дообучение существенно улучшило все LLM:

- исчезли parse errors;
- выросли классификационные метрики;
- заметно улучшились ROUGE-L и BERTScore;
- модели стали стабильнее соблюдать требуемый формат ответа.

Лучший общий результат среди LLM показала `Mistral-7B-Instruct-v0.3` после LoRA: `Accuracy = 0.802`, `macro-F1 = 0.747`, `ROUGE-L = 0.552`, `BERTScore = 0.920`.

Важно: LLM-модели немного уступили лучшей BERT-модели по классификации, но это не означает, что задача решена хуже. BERT-модели решали только классификацию, а LLM решали комплексную задачу: одновременно предсказывали класс и генерировали summary.

## Как запустить проект

Клонировать репозиторий:

```bash
git clone https://github.com/En-Donny/ODS-AI-NLP-FinalProject.git
cd ODS-AI-NLP-FinalProject
```

Создать виртуальное окружение:

```bash
python -m venv .venv
```

Активировать окружение:

```bash
# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate
```

Установить основные зависимости:

```bash
pip install pandas numpy scikit-learn openpyxl tqdm
pip install torch transformers datasets accelerate peft bitsandbytes trl evaluate
pip install rouge-score bert-score google-play-scraper sentencepiece protobuf
```

Сбор отзывов из Google Play:

```bash
python prepare_data/get_google_play_reviews.py
```

Очистка и подготовка отзывов к разметке:

```bash
python prepare_data/clean_reviews_for_annotation.py
```

Далее обучение и инференс запускаются через ноутбуки в директории `training/`.

## Основные выводы

1. На собственном русскоязычном датасете из 3194 отзывов получены сильные результаты для задачи классификации и краткой суммаризации.
2. Лучшей моделью для чистой классификации стала `ruRobertaLarge` с `macro-F1 = 0.801`.
3. Среди LLM лучшей стала `Mistral-7B-Instruct-v0.3` после LoRA-дообучения.
4. LoRA-дообучение значительно улучшило LLM по всем метрикам и устранило ошибки формата ответа.
5. Общий SOTA для всей области анализа отзывов приложений не заявляется, так как опубликованные работы используют другие языки, датасеты и постановки задач. Однако на данном датасете получены значимые результаты и построен воспроизводимый pipeline.
